The AIM Lifecycle

Author

Describe requirements in human-readable .intent files. High-level meaning, zero syntax.

Build

Agents synthesize production code from the intent. No hallucination, just logic.

Verify

Automated drift detection compares your intent to the code. Keep truth in sync.

Repair

Agents automatically restore alignment by fixing implementation or updating intent.

Operating Roles

AUTHOR

Writes the meaning. Defines success in plain .intent files.

"Initialize as an AIM Agent... in the Author role."
IMPLEMENTER

"Initialize as an AIM Agent... in the Implementer role."
VERIFIER

Checks the truth. Ensures code hasn't drifted from behavior.

REPAIRER

"Initialize as an AIM Agent... in the Repairer role."

Core Architecture

  • Shared Central Truth The /aim/ folder is the heart of your repo. It coordinates all agents and humans with one unambiguous mental model.
  • Strategic Model Usage Use reasoning models (Claude 3.5 Sonnet) for Authoring and Verification, and specialized coding models for Implementation.
  • Nested namespaces Materialize packages into standardized nested paths (/aim/game/snake/) automatically using registry-native agents.
weather.intent 
AIM: weather#intent@2.2

INTENT WeatherLookup {
  SUMMARY: "Real-time weather lookup."

  REQUIREMENTS {
    - "User enters a city name."
    - "UI displays temperature."
  }

  CONTRACT FetchWeather {
    INPUT { city: string required }
    ENSURES {
      - "CALLS WeatherAPI"
      - "RETURNS WeatherSnapshot"
    }
  }
}
