Intent-First Programming

Not another spec.md. AIM shifts AI software work from passing around context to operating on intent: structured, versioned artifacts agents can build from, verify against, and repair over time.

Beyond spec.md

A markdown file gives an agent context. AIM gives it explicit behaviors, contracts, and invariants that can survive implementation, verification, and repair.

Password Reset

From a sentence to enforceable behavior

spec.md
"Users can reset their password by email."
AIM
INTENT PasswordReset { ACTOR User ACTION RequestReset REQUIRES { "User owns the email address" } ENSURES { "Reset link is sent" } VERIFY { "Expired tokens cannot be reused" } }

Agents can build the flow and later verify ownership, token expiry, and single-use behavior.

Team Invites

From feature request to operating contract

spec.md
"Admins can invite members to a workspace."
AIM
INTENT TeamInvites { ACTOR Admin ACTION InviteMember ENSURES { - "Invite includes a role" - "Invite expires" - "Audit event is recorded" } }

Agents now have durable rules around authorization, role assignment, expiry, and auditability.

One Clean Change at a Time

AIM works as a commit-scoped loop. Click each step to see what you do and what the agent produces.

You

Commit pending work and start clean

Before running any agent, make sure the repo is in a committed, stable state — no uncommitted edits, no half-done features.

Why it matters

One source of truth, one clear diff

The agent sees exactly one version of the system. A clean baseline means a clean diff — and a clean diff means the agent can reason precisely about what changed and why.

You

Describe what the component should do

Tell the intent agent the actors, actions, rules, and invariants for the feature. No implementation details yet — just product meaning.

Intent Agent

Writes a structured intent file

Produces a versioned .intent file with ACTOR, ACTION, ENSURES, and VERIFY blocks — explicit contracts the coding agent can build from without guessing.

Initialize as an AIM v2.2 Agent (https://intentmodel.dev/brain.md) in the Intent Author role. I want to describe a new component.
You

Hand the intent file to the coding agent

Point the coding agent at the intent files. No further instructions needed — the intent layer has everything it needs to start building.

Coding Agent

Synthesizes code from intent

Builds working code where every function, type, and side effect maps back to an explicit contract in the intent file — not a guess from a free-form prompt.

Initialize as an AIM v2.2 Agent (https://intentmodel.dev/brain.md) in the Implementer role. I want to build a package.
You

Review the drift report

Look at what drifted between intent and implementation, approve or adjust intent, and let the repair agent restore alignment before anything leaves the branch.

Verifier + Repair Agent

Compares code vs intent, fixes drift

Runs a line-by-line comparison, produces a drift report, then restores alignment — updating intent or code to match, whichever is the source of truth.

Initialize as an AIM v2.2 Agent (https://intentmodel.dev/brain.md) in the Verifier role. Give me a drift report.
You

Commit intent and code together

Commit the intent files and implementation as one focused change. Push when ready.

What you get

A self-contained, reviewable commit

Intent, code, and alignment land together in one clean handoff — ready for a human reviewer, another agent, or a future repair pass with a better model.
