  REQUIREMENTS {
    - "Show temperature and location."

How it works

Works with whatever AI coding assistant you already use — Claude, Cursor, Copilot, or any chat-based agent. No new tools required.

1

Describe your feature

Tell your AI what you want to build in plain English — the actors, the rules, the edge cases. No special format needed yet.

2

AI writes the spec

The Author agent turns your description into a structured .intent file and saves it to your repo — a permanent source of truth.

3

Any agent builds from it

Point Claude, Cursor, or any coding assistant at the file. It reads the spec and builds — no more re-explaining the same context.

The difference

Without AIM, you paste requirements into every new chat and lose them when the session ends. With AIM, they live in your repo.

Without AIM — pasted into every chat
"Users reset their password by email. The link should expire after 24 hours. Tokens are single-use only. Log resets for auditing. The success page should redirect to login. Oh and admins..."

Repeated in every session. Lost when the chat closes. Inconsistent across agents.

With AIM — stored in your repo
INTENT PasswordReset {
  SUMMARY: "Email-based reset flow."
  REQUIREMENTS {
    - "User can request reset."
    - "Token expires after 24h."
    - "Single-use only."
    - "Log all resets."
  }
  CONTRACT {
    ENSURES { "Reset link is sent" }
    VERIFY  { "Single-use enforced" }
  }
}

Intent Author Start here

Turns your plain-English description into a structured .intent file. Use this first for any new feature or component.

Initialize as an AIM v2.2 Agent in the Intent Author role by reading your instructions at https://intentmodel.dev/brain.author.md. Read any existing .intent files for [Component Name]. Once ready, confirm you have successfully parsed the v2.2 specification and summarized the existing intent context (or noted its absence). Do not draft new intent yet.
Implementer

Synthesizes code and tests directly from your .intent files. Use this after the Author has created the spec.

Initialize as an AIM v2.2 Agent in the Implementer role by reading your instructions at https://intentmodel.dev/brain.implementer.md. Read the local intent in ./aim/[Component Name]. Once ready, confirm you have successfully parsed the v2.2 specification and summarized the requirements you are prepared to build. Do not build yet.
Verifier

Checks that your code still matches the intent after changes. Use this before merging to catch drift before it becomes a bug.

Initialize as an AIM v2.2 Agent in the Verifier role by reading your instructions at https://intentmodel.dev/brain.verifier.md. Map the intent in ./aim/ to the current implementation. Once ready, confirm you have successfully parsed the v2.2 specification and mapped the relationship between intent and implementation. Do not produce a drift report yet.
Repairer

Restores alignment when code and intent diverge — fixes the code to match the spec, or updates the spec to reflect what changed.

Initialize as an AIM v2.2 Agent in the Repairer role by reading your instructions at https://intentmodel.dev/brain.repairer.md. Read the intent and code for [Component Name]. Once ready, confirm you have successfully parsed the v2.2 specification and identified the drift context you are prepared to repair. Do not apply fixes yet.
