Contribute

Publish a Package

Follow these steps to contribute your component to the AIM registry. All submissions are reviewed via GitHub Pull Request.

Submission Workflow

1

Create a directory at registry/packages/<component>/.

2

Add exactly one #intent facet file: <component>.intent.

3

Add optional facet files referenced by the entry file via INCLUDES.

4

Add your package to the packages array in registry/index.json.

5

Ensure all files use valid AIM preambles (e.g., AIM: component#facet@2.2).

6

Submit a Pull Request to the GitHub repository.

Example Manifest

registry/index.json 
{
  "version": "1",
  "packages": [
    {
      "name": "weather",
      "version": "2.2",
      "entry": "registry/packages/weather/weather.intent"
    }
  ]
}

Validation Policy

Every PR is automatically validated for:

  • Correct directory structure
  • Header-to-path identity alignment
  • Version consistency
